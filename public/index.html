Create your free Google API credentials one-time, ~3 minutes — every step is one click away
  1. Create a project — any name works, it's private to you. Copy the project ID Google assigns (shown right under the name field) and paste it here so the next steps link straight into your project: Create project ↗
  2. Enable the Google Health API — the page that opens has a single Enable button. Enable API ↗
  3. Configure the consent screen — User type External; an app name and your email are enough. Consent screen ↗
  4. Add your own Google account as a test user — the most-missed step: without it Google refuses the sign-in with access_denied. Test users ↗
  5. Create the OAuth client — Application type: Desktop app when you run this dashboard on localhost; Web application when it's hosted on a website (register this page's URL — details in the hosting notes below). Create client ↗
  6. Copy the Client ID and Client Secret into the boxes below — done. Your data stays between Google and this browser.

Heads-up for later: because this is your own private Cloud project, Google shows an “unverified app” warning during sign-in. That's expected — it's your app; choose Continue.

Hosting & sign-in notes

Hosting this on the web (GitHub Pages etc.): create the OAuth client as type Web application and register this page's exact URL — https://you.github.io/dashboard/ — under both Authorized JavaScript origins (origin only) and Authorized redirect URIs.

Sign-in expires after ~7 days while your Cloud project's consent screen is in Testing mode. That's a Google limit for test apps — just sign in again.

Google shows an error page mid-sign-in? Some browser profiles (seen with Firefox extensions/site data) break Google's consent page itself. Try a private window or another browser — the dashboard works fine in any of them afterwards.

access_denied? Your Google account isn't on the consent screen's test-user list yet (step 4 above).
